# 네트워크 애플리케이션 계층: 비디오 스트리밍, CDN 및 소켓 프로그래밍

## 비디오 스트리밍 및 멀티미디어
### 비디오의 기본 특성
* 비디오 정의
    * 일정한 속도로 표시되는 이미지의 시퀀스(예: 24 images/sec)
* 비디오 코딩(Coding)
    * 중복성을 이용하여 비트 수를 줄임
    * 공간적 코딩(Spatial): 단일 이미지 내 중복되는 픽셀 정보를 압축
    * 시간적 코딩(Temporal): 이전 프레임과 다음 프레임 사이의 차이점만 전송
* 비트 전송률(Bit Rate)
    * CBR(Constant Bit Rate): 비디오 인코딩 속도가 고정됨
    * VBR(Variable Bit Rate): 화면의 복잡도에 따라 인코딩 속도가 변함

### 비디오 스트리밍의 과제
* 대역폭 가변성
    * 서버-클라이언트 간 대역폭은 네트워크 혼잡도에 따라 계속 변함
* 연속 재생 제약
    * 원래 타이밍에 맞춰 재생되어야 하며, 이를 위해 클라이언트 측 버퍼링(Buffering)이 필수적임
    * 버퍼는 네트워크 지연(jitter)을 보상하는 역할을 함

### DASH (Dynamic, Adaptive Streaming over HTTP)
* 서버
    * 비디오 파일을 여러 청크(chunk)로 나누고, 각 청크를 다양한 비트레이트로 인코딩하여 저장함
    * URL 목록이 담긴 Manifest 파일을 제공함
* 클라이언트
    * 자신의 가용 대역폭을 주기적으로 측정하여, Manifest를 참고해 최적의 비트레이트 청크를 요청함
    * 즉, 클라이언트가 상황에 맞춰 인코딩 속도와 서버를 결정하는 '지능'을 가짐

## 콘텐츠 전송 네트워크 (CDN)
### CDN의 필요성
* 단일 거대 서버(Mega-server) 방식은 단일 장애점(Single point of failure), 네트워크 혼잡, 먼 경로 등의 문제로 확장이 불가능함

### CDN 구축 전략
* Enter Deep
    * 액세스 네트워크 깊숙이 수많은 서버를 설치하여 사용자에게 최대한 밀착함 (예: Akamai)
* Bring Home
    * 적은 수의 대규모 클러스터를 액세스 네트워크 근처 POP에 설치함

### 사례 연구: Netflix
* OpenConnect
    * 자체 CDN 노드에 콘텐츠 복사본을 저장함
* Amazon Cloud 기능
    * 가입자 관리, 콘텐츠 탐색, Manifest 파일 제공 등의 제어 기능을 수행함
* 작동 흐름
    * 사용자가 비디오를 탐색하면 Manifest 파일을 받고, 클라이언트의 DASH 프로토콜이 최적의 CDN 서버를 선택해 스트리밍을 시작함

## 소켓 프로그래밍 (Socket Programming)
### 소켓의 개념
* 애플리케이션 프로세스와 전송 프로토콜 사이의 인터페이스(문, Door) 역할을 함

### UDP 소켓 (비연결형)
* 특징
    * 데이터 전송 전 핸드셰이킹 없음. 신뢰성이 없으며 패킷 순서가 바뀔 수 있음
* 동작
    * 보낼 때마다 목적지 IP와 포트 번호를 명시해야 함
* Python 코드 주요 함수
    * socket(AF_INET, SOCK_DGRAM): UDP 소켓 생성
    * sendto(): 데이터 전송
    * recvfrom(): 데이터 수신

### TCP 소켓 (연결 지향형)
* 특징
    * 신뢰성 있고 순서가 보장되는 바이트 스트림 전송 제공
* 동작
    * 통신 전 서버와 클라이언트 간의 연결 설정이 필요함
* 서버 소켓의 분리
    * Welcoming 소켓: 클라이언트의 최초 접속을 기다리는 소켓
    * Connection 소켓: accept() 호출 후 특정 클라이언트와 통신하기 위해 새로 생성되는 소켓
* Python 코드 주요 함수
    * socket(AF_INET, SOCK_STREAM): TCP 소켓 생성
    * connect(): 서버에 연결 요청
    * bind(), listen(), accept(): 서버 측 연결 대기 및 수락 과정
    * send(), recv(): 연결된 소켓을 통한 데이터 송수신
  
# 네트워크 전송 계층 (Transport Layer)

## 전송 계층의 역할 및 서비스
* 논리적 통신 제공
    * 서로 다른 호스트에서 실행되는 애플리케이션 프로세스 간의 논리적 통신을 제공함
* 송신자(Sender)의 동작
    * 애플리케이션 메시지를 세그먼트(segment) 단위로 분할하고, 헤더를 붙여 네트워크 계층(IP)으로 전달함
* 수신자(Receiver)의 동작
    * IP로부터 받은 세그먼트의 헤더를 확인하고 메시지를 추출하여, 소켓을 통해 적절한 애플리케이션으로 전달(역다중화)함

## 전송 계층 vs 네트워크 계층
* 네트워크 계층
    * 호스트(Host) 간의 통신을 담당함. (가정집 비유: 집과 집 사이의 배달)
* 전송 계층
    * 프로세스(Process) 간의 통신을 담당함. (가정집 비유: 집 안의 개별 구성원 간의 전달)
* 상호 관계
    * 전송 계층은 네트워크 계층의 서비스를 이용하며, 이를 강화하여 더 신뢰성 있는 서비스를 제공함

## 인터넷 전송 계층의 주요 프로토콜
인터넷 애플리케이션이 사용할 수 있는 두 가지 핵심 프로토콜이 있음

* TCP (Transmission Control Protocol)
    * 신뢰성 있는 전송: 데이터의 순서를 보장하고 손실 없는 전달을 책임짐
    * 제어 기능: 혼잡 제어(Congestion control)와 흐름 제어(Flow control)를 수행함
    * 연결 설정: 데이터를 주고받기 전 핸드셰이킹을 통한 연결 설정 과정이 필요함
* UDP (User Datagram Protocol)
    * 비신뢰적 전송: 데이터 전송 순서를 보장하지 않으며 손실 가능성이 있음
    * 경량화: IP의 "최선형(best-effort)" 서비스 모델을 최소한으로 확장한 형태임
* 제공하지 않는 서비스
    * 현재 인터넷 전송 계층 프로토콜은 지연 시간(delay) 보장이나 대역폭(bandwidth) 보장 서비스는 제공하지 않음

## 전송 계층의 주요 메커니즘 (학습 목표)
* 다중화 및 역다중화 (Mux/Demux)
    * 하나의 물리적 회선에서 여러 프로세스의 데이터를 구분하여 전달하는 기술
* 신뢰성 있는 데이터 전송 (RDT)
    * 비신뢰적인 하위 계층 위에서 어떻게 데이터 무결성을 유지할 것인가에 대한 원리
* 흐름 및 혼잡 제어
    * 송신자가 수신자나 네트워크가 감당할 수 없을 정도로 데이터를 빠르게 보내지 않도록 조절하는 방법
