# [Network] CH01. Computer Networks and the Internet

## 1.1 What is the Internet?

### 1.1.1 구성요소 관점 (Nuts-and-bolts view)

- **종단 시스템 (End Systems / Hosts):** 네트워크의 가장자리에 위치하여 애플리케이션을 실행하는 기기.
  - 데이터를 세그먼트(Segment)로 분할하고, 헤더를 붙여 **패킷(Packet)** 형태로 전송
  - 목적지에서 다시 원래 데이터로 재조립
- **패킷 스위치 (Packet Switches):** 도착한 패킷을 최적의 경로(출력 링크)로 전달하는 장치
  - **라우터(Router):** 네트워크 계층에서 경로 결정
  - **링크 계층 스위치(Link-layer Switch):** 로컬 네트워크 내 전달
- **통신 링크 (Communication Links):** 동축 케이블, 구리선, 광섬유, 라디오 스펙트럼 등 다양한 물리 매체
  - 전송 속도는 **R (Transmission Rate, bps)**로 측정
- **ISP (Internet Service Provider):** 종단 시스템이 인터넷에 접속할 수 있게 해주는 네트워크 서비스 제공자
  - 하위 ISP는 상위 ISP를 통해 연결되며, 상위 ISP들은 고속 광섬유 링크로 상호 연결됨
- **프로토콜 (Protocols):** TCP, IP, HTTP 등 정보 송수신을 제어하는 약속

### 1.1.2 서비스/인프라 관점 (Service view)

- **분산 애플리케이션:** 웹, 이메일, 게임 등 여러 종단 시스템 간 데이터를 주고받는 시스템
- **소켓 인터페이스 (Socket Interface):** 한 프로그램이 다른 프로그램으로 데이터를 전달하도록 요청하는 방식(API)을 명시

### 1.1.3 프로토콜 (Protocol)의 정의

- **정의:** 통신 주체 간에 교환되는 **메시지의 형식과 순서**, 그리고 메시지 전송/수신 및 이벤트 발생 시 **취해지는 행동**을 정의한 규칙.

## 1.2 The Network Edge

### 1.2.1 액세스 네트워크 (Access Networks)

- **DSL (Digital Subscriber Line):** 기존 전화선을 이용. DSLAM을 통해 데이터와 음성 신호를 분리.
- **케이블 인터넷:** TV 케이블 인프라(HFC, 광동축 혼합망)를 사용. 공유 매체 특성을 가짐.
- **FTTH (Fiber to the Home):** CO(국사)에서 가정까지 직접 광섬유 연결.
  - **PON (Passive Optical Network):** 스플리터를 통해 여러 가정이 하나의 광섬유를 공유하는 구조 (ONT - Splitter - OLT).
- **LAN (Local Area Network):** 이더넷(Ethernet)이나 Wi-Fi(AP)를 통해 기업/가정 내부를 연결.

### 1.2.2 물리 매체 (Physical Media)

- **유도 매체 (Guided Media):** 고체 매체를 따라 신호 전달.
  - **꼬임쌍선 (UTP):** LAN에서 주로 사용.
  - **동축 케이블:** 높은 데이터 전송률, 공유 매체.
  - **광섬유:** 매우 높은 속도, 전자기 간섭에 강함, 해저/장거리 링크용.
- **비유도 매체 (Unguided Media):** 대기나 우주 공간으로 전파.
  - 지상 라디오 채널, 위성 채널.

## 1.3 The Network Core

### 1.3.1 패킷 교환 (Packet Switching)

- **Store-and-Forward:** 라우터가 패킷의 모든 비트를 수신한 후에야 다음 링크로 전송을 시작함.
  - 전송 지연 시간: $L/R$ (L: 패킷 길이, R: 전송 속도)
  - N개 링크(N-1개 라우터) 통과 시 지연: $d_{end-to-end} = N \times (L/R)$
- **큐잉 지연 및 패킷 손실:** 링크가 사용 중이면 출력 버퍼(큐)에서 대기하며, 버퍼가 가득 차면 패킷이 유실됨.
- **포워딩 테이블:** 목적지 주소를 기반으로 패킷을 어느 출력 링크로 보낼지 결정.

### 1.3.2 회선 교환 (Circuit Switching)

- 통신 세션 동안 자원(대역폭)을 **예약(Reserve)**하여 전용 경로를 확보.
- 구현 방식:
  - **FDM (주파수 분할):** 주파수 대역을 나누어 할당.
  - **TDM (시간 분할):** 시간을 슬롯으로 나누어 할당.
- **특징:** 지연이 일정하지만, 비활동 시간(Silent period) 동안 자원이 낭비됨.

### 1.3.3 패킷 교환 vs 회선 교환

- **패킷 교환의 승리:** 더 많은 사용자 수용 가능. (예: 1Mbps 링크에서 10%만 활동하는 사용자 상정 시, 패킷 교환은 회선 교환보다 훨씬 많은 사용자를 동시에 수용 가능)

### 1.3.4 네트워크의 네트워크 (A Network of Networks)

- **Tier-1 ISP:** 전 세계를 연결하는 최상위 사업자.
- **IXP (Internet Exchange Point):** ISP들이 모여 직접 데이터를 교환하는 접점 (비용 절감).
- **Peering:** ISP 간에 비용 지불 없이 직접 데이터를 주고받는 관계.

## 1.4 지연(Delay), 손실(Loss), 처리율(Throughput)

### 1.4.1 4가지 노드 지연 (Nodal Delay)

1.  **노드 처리 지연 ($d_{proc}$):** 헤더 조사, 에러 체크, 출력 링크 결정.
2.  **큐잉 지연 ($d_{queue}$):** 출력 링크가 비기를 기다리는 시간. (네트워크 혼잡도에 따라 가변적)
3.  **전송 지연 ($d_{trans}$):** 패킷의 모든 비트를 링크로 밀어내는 시간. ($L/R$)
4.  **전파 지연 ($d_{prop}$):** 링크 매체를 통해 신호가 물리적으로 이동하는 시간. ($Distance / Speed\_of\_light$)

$$d_{nodal} = d_{proc} + d_{queue} + d_{trans} + d_{prop}$$

## 3) AI가 알려주는 헷갈리는 포인트

### 1. 전송 지연 vs 전파 지연 (가장 많이 틀리는 부분)

이 두 개념은 이름이 비슷해서 계산 문제에서 자주 섞입니다.

- **전송 지연 ($d_{trans} = L/R$):** "패킷을 링크로 **밀어내는** 시간"입니다. 비트를 고속도로 톨게이트에서 한 대씩 내보내는 속도라고 생각하세요. 자동차(비트)가 많거나 톨게이트(대역폭)가 좁으면 늘어납니다.
- **전파 지연 ($d_{prop} = d/s$):** "비트가 링크를 타고 **이동하는** 시간"입니다. 톨게이트를 빠져나온 차가 다음 목적지까지 달리는 시간입니다. 오로지 **거리**와 **매체의 물리적 속도(광속 등)**에만 영향을 받습니다.
  - **헷갈림 포인트:** "대역폭(R)이 커지면 패킷이 목적지에 도착하는 전체 시간이 줄어드는가?" -> 예, 전송 지연이 줄어들기 때문입니다. 하지만 전파 지연 자체는 변하지 않습니다.

### 2. 회선 교환 vs 패킷 교환의 '효율성' 해석

단순히 "패킷 교환이 더 좋다"고 외우면 안 됩니다. 상황에 따라 다릅니다.

- **패킷 교환이 유리한 이유:** "통계적 다중화(Statistical Multiplexing)" 때문입니다. 모든 사용자가 동시에 데이터를 보내지 않는다는 점을 이용해, 실제 회선 용량보다 더 많은 사용자를 수용할 수 있습니다. (전달해주신 10% 활동 사용자 예시가 핵심입니다.)
- **회선 교환이 유리한 이유:** 자원을 예약(Reserve)하기 때문에 **품질 보장(QoS)**이 확실합니다. 실시간 스트리밍이나 중요한 화상 회의처럼 지연 시간의 변화(Jitter)가 없어야 하는 서비스에 강점이 있습니다.
  - **헷갈림 포인트:** 패킷 교환은 자원을 예약하지 않기 때문에 사용자가 갑자기 몰리면 **큐잉 지연(Queuing Delay)**과 **패킷 손실(Packet Loss)**이 발생할 수 있다는 단점을 명확히 인지해야 합니다.

### 3. 계층별 장비의 역할 (L2 vs L3)

노트에서 패킷 스위치 부분에 "라우터"와 "링크 계층 스위치"를 언급하셨는데, 이 둘의 차이가 중요합니다.

- **라우터 (L3 장비):** **IP 주소**를 보고 목적지를 찾습니다. 네트워크와 네트워크를 연결하는 '교차로' 역할을 합니다.
- **스위치 (L2 장비):** **MAC 주소**를 보고 같은 네트워크 안에서 전달합니다.
  - **헷갈림 포인트:** "인터넷 코어(Network Core)"에서 주로 말하는 '패킷 스위칭'은 대부분 **라우터**들 간의 동작을 의미합니다.

### 4. 계산 문제의 단위 (bps vs Bytes)

노트 마지막에 전송 지연 $L/R$을 적으셨는데, 실제 계산 시 단위를 조심하세요.

- **L(패킷 길이):** 보통 **Byte** 단위로 주어집니다.
- **R(전송 속도):** 보통 **bps(bits per second)** 단위로 주어집니다.
- **주의:** 계산할 때 반드시 **1 Byte = 8 bits**로 변환해서 단위를 맞춰야 정확한 초($sec$) 단위 지연 시간이 나옵니다.
