# 1.1 인터넷이란 무엇인가?

1. **인터넷의 구성요소**: 하드웨어와 소프트웨어 구성요소
2. **네트워킹 인프라스트럭처**: 분산 애플리케이션에 서비스를 제공

## 1.1.1 구성요소로 본 인터넷

- 호스트 (종단 시스템): 모든 컴퓨팅 장치
- 경로 (네트워크)
: 송신 종단 시스템 -> 수신 종단 시스템까지 거쳐온 일련의 통신 링크와 패킷 스위치.
    - 통신 링크: 물리 매체
        - 동축케이블, 구리선, 광케이블, 라디오 스펙트럼 등
        - 다양한 전송률
    - 패킷 스위치
    : 입력 통신 링크의 하나. 도착하는 패킷을 받아 출력 통신 링크의 하나로 패킷 전달
        - 라우터: 네트워크 코어에서 사용
        - 링크 계층 스위치: 접속 테트워크에서 사용
- 전송률
    - bps: bit per second. 초당 비트 수
- 패킷
: 네트워크 데이터 단위
    - 세그먼트: 데이터를 나눈 조각
    - 헤더: 세그먼트에 붙는 메타 데이터
- ISP(Internet Service Provider)
: 종단 시스템은 ISP를 통해 인터넷에 접속
    - 각 ISP는 패킷 스위치와 통신 링크로 이루어진 네트워크
    - 네트워크 접속 제공
    - CP(content provider)에게 인터넷 접속 제공
- 하위 ISP -> 상위 ISP
    - 하위 계층 ISP: 상위 계층 ISP를 통해 연결
    - 상위 계층 ISP: 광 링크로 연결된 고속 라우터로 직접 연결
- 프로토콜
    - TCP/IP
    - IP 프로토콜: 라우터와 종단 시스템 사이에서 송수신되는 패킷 포맷
- IETF(Internet Engineering Task Force): 인터넷 표준 개발 기구
    - RFC(requests for comment): IETF 표준 문서

## 1.1.2 서비스 측면에서 본 인터넷

## 1.1.3 프로토콜이란 무엇인가?

### 사람 프로토콜

: 명확하게 송수신된 메시지와 이러한 메시지가 송수신될 때나 다른 상황이 발생했을 때 취하는 행동

### 네트워크 프로토콜

: 둘 이상의 통신 개체 간에 교환되는 메시지 포맷과 순서뿐만 아니라, 메시지의 송수신과 다른 이벤트에 따른 행동들 정의

- 통신하는 둘 이상의 원격 개체가 포함된 인터넷 활동 제어
- 컴퓨터 사이 선로상의 비트 흐름 제어
    - 혼잡 제어 프로토콜
        - 종단 시스템에 존재
        - 송수신자 간에 전송되는 패킷 전송률 조절
    - 라우터
        - 출발지에서 목적지까지 패킷 경로로 설정

## 1.2.1 접속 네트워크

- 접속 네트워크
: 종단 시스템이 경로상에서 만나는 첫 번째 라우터(가장자리 라우터)

### 가정 접속: DSL, 케이블, FTTH, 5G 고정 무선

- 가장 널리 보급: DSL, 케이블
- DSL(digital subscriber line)
: *지역 전화 회사(telco)*로부터 DSL 인터넷 접속 서비스 제공받음
    - 텔코는 유선 로컬 전화 서비스 제공
    - 텔코가 ISP가 됨
    - 각 고객의 **DSL 모뎀** <-> 텔코의 지역 중앙국(CO)의 **DSLAM**
        - 데이터 교환하기 위해 기존 전화 회선 이용
        - DSL 모뎀: 디지털 데이터 -> 고주파 신호
        - DSLAM: 아날로그 신호 -> 디지털 포맷
    - 데이터와 전화 신호는 서로 다른 주파수 대역에서 인코딩
        1. 고속 *다운스트림 채널*: 50kHz ~ 1MHz 대역
        2. 중간 속도 *업스트림 채널*: 4 ~ 50kHz 대역
        3. 일반적인 양방향 *전화 채널*: 0 ~ 4kHz 대역
        - 고객 쪽의 splitter: 도착하는 데이터와 전화 신호 분리 -> 데이터 신호를 DSL 모뎀으로 전송
        - DSLAM: 데이터와 전화 신호 분리 -> 데이터를 인터넷으로 송신
    - DSL 표준
        - 다운스트림: 24Mbps, 52Mbps 둥
        - 업스트림: 3.5Mbps, 16Mbps 등
        => 비대칭 접속 방식: 다운/업스트림의 속도가 다름
- 케이블 인터넷 접속
: *케이블 TV 회사*의 기존 케이블 TV 인프라스트럭처 이용
    - 광케이블: 케이블 head end(케이블망 중앙국)를 이웃 레벨 junction에 연결
    => junction은 개별 가정/아파트에 동축케이블로 연결
    - HFC(hybrid fiber coax)라고 부름(광케이블, 동축케이블 모두 채택)
    - 케이블 모뎀: 외장형 장치. 이더넷 포트를 통해 가정 PC에 연결
        1. 다운스트림 채널
        2. 업스트림 채널
    - CMTS: DSLAM과 유사한 기능 제공
    - 공유 방송 매체
        - 헤드엔드가 보낸 모든 패킷이 모든 링크의 다운스트림 채널을 통해 모든 가정으로 전달
        - 가정에서 보낸 모든 패킷은 업스트림 채널을 통해 헤드엔드로 전달
        => 여러 사용자가 다운스트림 채널에서 다른 비디오 파일을 동시에 수신 -> 실제 수신율이 다운스트림 전송률보다 상당히 감소
        => 분산 다중 접속 프로토콜 필요
- FTTH(fiber to the home)
: CO부터 가정까지 *직접 광섬유 경로 제공*
    - CO부터 가정까지 광신호 분배 기술
        1. 다이렉트 광섬유: 각 가정으로 CO에서 하나의 광섬유 제공
        2. 스플리팅: 가정에 가까운 곳까지 하나의 광섬유 -> 고객별 분리
            - AON(active optical network): 교환 이더넷
            - PON(passive optical network)
                - 가정마다 ONT 보유
                - ONT는 지정된 광섬유로 이웃 스플리터에 연결
                - 스플리터는 여러 가정을 하나의 공유 광섬유로 결합
                - 공유 광섬유는 텔코의 CO의 OLT에 연결
                - OLT: 광신호와 전기 신호 변환. 텔코 라우터를 통해 인터넷에 연결
                - 사용자는 홈 라우터를 ONT에 연결, 인터넷에 접속
                - OLT -> 스플리터로 송신된 모든 패킷은 스플리터에서 복제

### 기업(그리고 가정) 접속: 이더넷과 와이파이

- LAN: 종단시스템을 가장자리 라우터에 연결하는 데 사용
- 이더넷
    - 이더넷 스위치에 연결하기 위해 꼬임쌍선 이용
    - 이더넷 스위치는 더 큰 인터넷으로 연결
    - 사용자: 100Mbps ~ 10Gbps 속도로 접속
    - 서버: 1Gbps ~ 10Gbps 속도로 접속
- 와이파이
: 무선 랜 접속
    - AP(access point)로 패킷 송/수신

### 광역 무선 접속: 3G와 LTE 4G와 5G

- 무선 인프라스트럭처
- 이동 전화망 기지국: 패킷 송수신

## 1.2.2 물리 매체

- 유도 매체: 광섬유 케이블, 꼬임쌍선, 동축 케이블 같은 견고한 매체 -> 파형 유도
- 비유도 매체: 무선 랜, 디지털 위성 채널처럼 대기와 야외 공간으로 -> 파형 전파

# **1.3 네트워크 코어**

## 1.3.1 패킷 교환

- 패킷 스위치: R비트/초로 송신
- 패킷: L비트
    ⇒ 전송 시간: L/R
- 저장 후 전달 전송 방식
    - 패킷의 첫 비트를 보내기 전에, 전체 패킷을 받아야 함
    - 목적지 노드까지 N개의 링크, N-1개의 라우터, L비트 패킷 1개
        - d종단 간 지연 = N * L/R
- 큐잉 지연: 출력 버퍼에서 대기하는 시간
- 패킷 손실: 출력 버퍼가 꽉 찬 경우 패킷 폐기
- 도착 패킷 전송률 > 출력 링크 전송률: 혼잡 발생 → 출력 버퍼 큐잉
- 포워딩 테이블: 목적지 주소 → 출력 링크 매핑
    - 라우팅 프로토콜: 각 라우터로부터 각 목적지까지의 최단 경로 결정 → 포워딩 테이블 설정

## 1.3.2 회선 교환

링크와 스위치의 네트워크를 통해 데이터 이동시키는 방식

1. 회선 교환: 자원 예약 O
2. 패킷 교환: 자원 예약 X
- 회선 교환 네트워크
    - 경로상에 필요한 자원(버퍼, 링크 전송률)이 통신 세션 동안 확보/예약됨
    - 패킷 교환 네트워크는 자원 예약 X
- 전화망: 연결 유지 필요
    - ‘연결’ = 회선
- 링크가 회선으로 분할하는 방법
    1. 주파수 분할 다중화(FDM): 주파수 대역
    2. 시분할 다중화(TDM): 일정 주기의 프레임 → 고정된 수의 시간 슬롯
        1. 시간 슬롯 1개를 연결에 할당: 한 데이터를 보낼 떄 할당된 슬롯 내에서만 전송 가능
        2. TDM 회선 전송률
            1. 한 슬롯 내 비트 수 * 프레임 전송률(단위 시간에 보내는 프레임 수)
            2. 링크 전송률 / 슬롯의 개수
- 패킷 교환 vs 회선 교환
    - 패킷 교환 옹호자
        - 전송 용량 공유에서 더 효율적
        - 더 간단하고 효율적, 구현 비용 적음
        - 회선 교환은 비활용 기간에는 놀게 됨
    - 회선 교환 옹호자
        - 패킷 교환이 가변적이고 전송 지연이 예측 불가

## 1.3.3 네트워크의 네트워크

접속 ISP들끼리도 서로 연결되어야 함

1. 네트워크 구조 1: 모든 접속 ISP를 하나의 글로벌 통과 ISP와 연결
2. 네트워크 구조 2 (2계층 구조):  수십만 개의 접속 ISP와 다중의 글로벌 ISP
3. 네트워크 구조 3 (다중계층구조): 접속 ISP - 지역 ISP - 1계층 ISP
4. 네트워크 구조 4: 네트워크 구조 3 + PoP, 멀티홈, 피어링, IXP
    1. PoP(points of presence)
        1. 접속 ISP를 제외한 모든 계층에 존재
        2. 제공자 네트워크에 존재하는, 고객 ISP가 제공자 ISP와 연결되는 라우터 그룹
    2. 멀티홈
        1. 둘 이상의 제공자 ISP에 연결
    3. 피어링
        1. 고객 ISP들끼리 통신 (제공자 ISP 지불 요금 줄이기 위해)
    4. IXP(Internet Exchange Point)
        1. 다중 ISP들이 서로 피어링할 수 있는 만남의 장소
5. 네트워크 구조 5: 네트워크 구조 4 + 콘텐츠 제공자 네트워크
    - 콘텐츠 제공자 네트워크
        - 자신의 네트워크 직접 구축
        - 주로 IXP 내에 위치

# 1.4 패킷 교환 네트워크에서의 지연, 손실과 처리율

## 1.4.1 패킷 교환 네트워크에서의 지연 개요

업스트림 노드 → 라우터 A → 라우터 B

### 처리 지연

패킷 헤더 조사 & 출력 링크 결정 + 오류 조사

### 큐잉 지연

앞선 대기 중인 패킷 수에 따라 결정

### 전송 지연

패킷의 길이(L bit) / 링크의 전송률(R bps)

### 전파 지연

두 라우터 사이의 거리(d) / 링크의 전파 속도(s)

### 전체 노드 지연
처리 지연 + 큐잉 지연 + 전송 지연 + 전파 지연
